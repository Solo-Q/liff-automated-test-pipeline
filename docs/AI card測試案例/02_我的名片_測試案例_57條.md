### ⚠️ 待釐清匯總（需 PM/RD 回覆）

- 未上傳頭圖時 Flex Message 的顯示樣式（預設圖？背景圖＋icon？完全空白？）
- 姓名、職稱、公司名稱等文字欄位的長度上限與截斷規則
- 手機號碼與 Email 欄位的格式驗證規則（前端/後端是否阻擋錯誤格式）
- 「傳送郵件」是否需預設主旨/內容，以及其文案
- 主要名片變更的併發策略（多裝置同時操作時最終一致性定義）
- 「公開名片資訊」狀態變更對已載入 Flex 的即時性（是否需動態更新按鈕顯示）
- 以連結分享的實際格式細節（應完全以外部規格文件為準）

### 附錄：例外情境預設預期結果（需與 RD 確認）

| 例外類型 | 預設預期結果 |
|----------|-------------|
| API 失敗（4xx / 5xx） | 顯示錯誤提示，操作不執行，使用者可重試 |
| 網路逾時 | 顯示「請稍後再試」，不重複寫入資料 |
| 連續點擊 / 重複提交 | 僅執行一次，防重複提交（按鈕 disabled 或 debounce） |
| 網路中斷（操作中途） | 顯示網路錯誤提示，恢復網路後可重試；已寫入的資料不重複寫入 |
| 多人同時操作同一筆資料 | 各自操作成功，後送出的結果以最終狀態為準，不會互相覆蓋或產生錯誤 |
| 流程中途離開再回來 | 回到流程起點或上次停留頁面，不殘留錯誤狀態 |
