## Mock / 測試資料建議

- **測試帳號與方案**
  - 免費方案帳號：至少 3 個（分別模擬「剩餘 50 次」「剩餘 1 次」「剩餘 0 次」）
  - 加購方案帳號：至少 1 個（含「已用完基本額度 + 加購額度未用完」）
  - 多帳號交叉測試：一組用於建立既有名片，一組作為「已註冊使用者」電話來源。

- **名片圖片樣本**
  - 正常清晰且欄位完整的名片（含姓名、手機、Email、公司、職稱、LINE ID、公司電話）
  - 僅部分欄位（只含姓名 + 手機 + 公司）
  - 大量非結構化文字（行銷文案、活動資訊）
  - 兩組以上手機號碼
  - 無手機號碼
  - 模糊、反光、裁切不完整的名片

- **API / 環境控制**
  - 可設定掃描 API 回應：成功、4xx、5xx、timeout
  - 可模擬網路中斷（送出前、送出後）
  - 可調整伺服器日期以模擬計費月切換與掃描次數重置。

## Smoke Test 清單（30 分鐘內）

- **SC-SMOKE-01 掃描名片 Happy Path（P0，預估 5 分鐘）**
  - 流程：首次授權相機 → 拍攝清晰名片 → 自動帶入欄位 → 建立名片成功
  - 目的：驗證核心掃描與名片建立流程可用。

- **SC-SMOKE-02 已授權相機再次掃描（P0，預估 3 分鐘）**
  - 流程：相機已授權 → 直接進入掃描頁 → 掃描並建立名片
  - 目的：驗證權限狀態被正確記錄且不重複彈窗。

- **SC-SMOKE-03 掃描次數扣減與用罄提示（P0，預估 10 分鐘）**
  - 流程：使用剩餘 1 次掃描 → 完成一次掃描 → 確認次數變 0 → 再次進入掃描頁面
  - 目的：驗證扣次與用罄行為正確。

- **SC-SMOKE-04 API 失敗處理（P0，預估 5 分鐘）**
  - 流程：模擬掃描 API 5xx → 嘗試掃描 → 檢查錯誤提示與不扣次
  - 目的：驗證失敗情境下不會誤扣次且不建立錯誤資料。

- **SC-SMOKE-05 已有名片手機號碼阻擋（P0，預估 7 分鐘）**
  - 流程：掃描一張手機號碼與既有名片重複的名片 → 嘗試建立
  - 目的：驗證手機唯一性與錯誤提示。

> 以上 Smoke Test 依序執行，總預估時間約 30 分鐘內。
