# 03_收入流程_測試案例

## 模組說明

收入流程負責在使用者點擊「收入名片夾」相關入口後，依照：
- 使用者是否已註冊
- 是否為「透過分享名片首次啟動註冊」情境
- 對方的「自動同意收入申請」設定（ON / OFF）
- 使用者的「自動傳送名片」設定（ON / OFF）
- 進入方式（展開查看 → 再點收入 / 直接在 Flex 點收入）

決定：
- 是否先進入註冊流程
- 是否將對方名片直接收入到名片夾
- 是否同時將自己的名片自動傳送給對方
- 顯示哪一段 Toast 文案
- 是否推送何種通知（待收入 / 收入申請 / 申請通過 / 首次收入相關推播）

以下測試案例皆以「一列一個主要驗證目標」原則拆分。

## Mock / 測試資料建議

- **測試帳號與設定**
  - **己方帳號 A**：已註冊，提供可調整「自動傳送名片」ON/OFF 的測試帳號。
  - **對方帳號 B**：已註冊，提供可調整「自動同意收入申請」ON/OFF 的測試帳號。
  - **新用戶帳號 C**：尚未註冊，用於驗證「未註冊 → 註冊 → 首次收入/申請」情境。
  - 覆蓋以下設定組合矩陣：  
    - B 自動同意 = ON / OFF × A 自動傳送 = ON / OFF（共 4 種組合）。

- **Flex Message 與名片內容**
  - 至少準備一組可反覆發送的名片 Flex，內容固定，僅用於驗證收入流程與通知，不混入其他功能。
  - 若有「分享名片啟動註冊」專用 Flex 模板，需提供對應測試入口。

- **系統側 Mock**
  - 收入 API（寫入名片夾）需可模擬：
    - 正常成功
    - 4xx 錯誤（參數錯誤）
    - 5xx 伺服器錯誤
    - 逾時與網路中斷
  - 通知推送服務需可觀察實際推送 payload 或以測試機制檢視收/發結果。

## 覆蓋率檢查清單

- **進入點與註冊狀態**
  - [ ] 方式一：Flex 展開 → 查看名片 → 收入（已註冊 / 未註冊）
  - [ ] 方式二：Flex 直接點收入（已註冊 / 未註冊）
  - [ ] 透過分享名片啟動註冊流程後的首次收入/申請（對方自動同意 ON / OFF）

- **設定組合矩陣**
  - [ ] 對方自動同意=ON × 使用者自動傳送=OFF（收入但不傳送自己名片）
  - [ ] 對方自動同意=ON × 使用者自動傳送=ON（收入 + 自動傳送 + 待收入通知）
  - [ ] 對方自動同意=OFF × 使用者自動傳送=OFF（收入申請流程）
  - [ ] 對方自動同意=OFF × 使用者自動傳送=ON（收入申請 + 己方名片傳送行為待釐清）

- **Toast 與通知**
  - [ ] 每一種設定組合下的 Toast 文案與顯示時機皆有測試案例覆蓋。
  - [ ] 「待收入通知」、「收入申請通知」、「申請通過通知」三種通知的發送條件與內容皆有對應案例。
  - [ ] 首次收入 / 首次名片申請的推播訊息（兩種文案）皆有驗證。

- **例外與錯誤處理**
  - [ ] API 失敗（4xx / 5xx）對收入結果與通知的影響。

  - [ ] 快速連點 / 重複操作的防重複提交。

## Smoke Test 清單（約 30 分鐘內）

- **ST-03-01：已註冊＋自動同意＋自動傳送（核心 Happy Path）**
  - 步驟：以 TC-03-001 的方式從 Flex 直接點擊「收入名片夾」。
  - 驗證：對方名片加入名片夾、顯示正確 Toast、對方收到「待收入通知」。
  - 預估時間：3 分鐘。

- **ST-03-02：已註冊＋自動同意＋不自動傳送**
  - 步驟：以 TC-03-002 的方式從查看名片頁面點擊「收入名片夾」。
  - 驗證：名片加入名片夾、Toast 文案無「並同步將您的名片分享給對方」、不推送待收入通知。
  - 預估時間：3 分鐘。

- **ST-03-03：已註冊＋對方不自動同意（收入申請流程）**
  - 步驟：以 TC-03-006 的方式送出收入申請，並以對方身分同意。
  - 驗證：使用者看到「已送出申請！正在等待對方同意」Toast、對方收到「收入申請」通知、同意後使用者收到「XXX 已通過你的收入申請」通知且名片出現在名片夾。
  - 預估時間：7 分鐘。

- **ST-03-04：透過分享名片啟動註冊（對方自動同意=ON）**
  - 步驟：依 TC-03-014 進行（分享名片 → 收入 → 註冊完成後自動收入）。
  - 驗證：註冊完成後自動收入成功，並收到「恭喜完成首次名片收入！」推播。
  - 預估時間：8 分鐘。

- **ST-03-05：透過分享名片啟動註冊（對方自動同意=OFF）**
  - 步驟：依 TC-03-015 進行（分享名片 → 收入 → 註冊 → 自動送出申請）。
  - 驗證：註冊完成後不直接收入，而是送出收入申請，並收到「恭喜完成第一次名片申請！」推播，對方同意後名片才出現在名片夾。
  - 預估時間：8 分鐘。

> 總預估時間：約 29 分鐘，適合作為部署後快速驗證流程。

- **S-IN-01：已註冊使用者收入－對方自動同意 ON（Happy Path）**（預估 5 分鐘）  
  - 以已註冊帳號從 Flex 直接點「收入名片夾」，對方自動同意 ON，確認名片夾成功新增該名片且顯示正確 toast。

- **S-IN-02：已註冊使用者收入－對方自動同意 OFF（申請流程）**（預估 7 分鐘）  
  - 以已註冊帳號對自動同意 OFF 的對方執行收入，確認 A 端顯示「已送出申請」，B 端收到收入申請通知，B 同意後 A 名片夾新增名片並收到通過通知。

- **S-IN-03：未註冊使用者透過分享名片啟動註冊＋首次收入（對方自動同意 ON）**（預估 8 分鐘）  
  - 透過分享名片入口啟動註冊，完成手機驗證與名片建立後，確認系統自動續接收入並推播「恭喜完成首次名片收入！」，名片夾中出現第一張名片。

  - 對同一名片先成功收入一次，再次點擊收入入口，確認不會重複新增名片；對同一入口連續快速點擊，確認僅產生一筆收入或申請。

### 例外情境預設預期結果（附錄－收入流程共用）

> PRD 未明確定義收入流程之例外處理時，以下為本文件中使用的預設值，並在各案例預期結果欄標記「⚠️ 預設值，需與 RD 確認」。

| 例外類型 | 預設預期結果 |
|----------|-------------|
| API 失敗（4xx / 5xx） | 顯示錯誤提示（如「操作失敗，請稍後再試」），該次收入或申請不生效，使用者可重新嘗試 |
| 網路逾時 | 顯示「請稍後再試」，不重複寫入資料或重複送出申請 |
| 連續點擊／重複提交 | 僅執行一次收入或建立一次申請，其餘點擊在前端或後端被忽略（按鈕 disabled 或 debounce） |
| 網路中斷（操作中途） | 顯示網路錯誤提示，恢復網路後可再次從入口重新嘗試；已成功寫入的資料不重複寫入 |
| 多人同時操作同一張名片 | 各申請人各自產生獨立申請；最終以每筆申請的處理結果為準，不會互相覆蓋或產生錯誤 |
| 流程中途離開再回來 | 再次從入口啟動時可重新開始收入／申請流程，不殘留半成品狀態或卡死畫面 |
